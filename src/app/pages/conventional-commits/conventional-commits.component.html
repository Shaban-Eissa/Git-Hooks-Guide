Write Better Commit Messages

A specification for adding human and machine-readable meaning to commit messages. Follow this guide to create structured, meaningful commit history.

Commit Message Format 

type(scope): description

[optional body]

[optional footer(s)]

Type

Describes the kind of change

Required

Scope

Section of the codebase

Optional

Description

Short summary in present tense

Required

Real-World Examples

Feature with Breaking Change 

feat(api)!: change login API response format

BREAKING CHANGE: The login API now returns a JWT token instead
of a session cookie.

- Remove cookie-based authentication
- Add JWT token validation
- Update client-side token storage

Bug Fix with Issue Reference 

fix(dashboard): resolve data loading issue

The dashboard wasn't loading data when filters were applied.
This fixes the API query parameter formatting.

Fixes #123

Refactor with Multiple Scopes 

refactor(auth,api): centralize error handling

Move error handling logic to a shared service to ensure
consistent error messages across the application.
